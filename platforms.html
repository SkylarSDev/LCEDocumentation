Platform Support

Overview

The LCE codebase supports 6 platforms through a shared Common/ code layer with platform-specific implementations. Each platform provides its own network manager, social/achievements integration, audio system, UI/input handling, and asset management. 4J Studios' proprietary libraries abstract many platform differences.

Platform Abstraction

The engine is architected around a shared core with per-platform specialization directories:

  • Common/ folder contains all shared cross-platform code
  • Each platform has its own directory with implementations of abstract interfaces
  • 4J Libraries (4J_Input, 4J_Storage, 4J_Profile, 4J_Render) provide the abstraction layer
  • Build system selects platform-specific code via preprocessor defines and CMake configuration

Per-Platform Implementations

Each platform implements the following subsystems:

Network Manager Online services, matchmaking
Social Integration Friends, achievements, leaderboards
Audio Initialization Miles Sound System per-platform
Input Handling Controller, keyboard, touch
Asset/DLC Management Platform store integration
Save System Platform storage APIs

Windows 64-bit (LCEMP)

Windows 64-bit
Minecraft.Client/Windows64/
  • Target: Win32 x64 Executable
  • Preprocessor: _WINDOWS64
  • Rendering: DirectX 11 via 4J_Render_PC, d3d11.lib
  • Input: KeyboardMouseInput.cpp for keyboard/mouse, XInput9_1_0.lib for controllers
  • Audio: Miles Sound System (mss64.lib)
  • UI: Iggy/Scaleform (iggy_w64.lib)
  • Libraries: Windows64/4JLibs/, Windows64/Miles/, Windows64/Iggy/, Windows64/Sentient/
  • Unique Features: LAN multiplayer (LCEMP addition), fullscreen toggle, keyboard/mouse support, gamma correction
  • Build: CMake + Visual Studio 2012 solution

Xbox 360

Xbox 360
Minecraft.Client/Xbox/
  • Target: Xbox 360 console

Key Features

  • Kinect support for gesture-based interaction
  • Xbox Live integration (multiplayer, achievements, leaderboards)
  • Custom audio system with console-specific sound banks
  • Split-screen multiplayer up to 4 players
  • Title Update system for content patches
  • Libraries: Xbox/4JLibs/, Xbox/Sentient/
Notes: Original launch platform for LCE. Resource-constrained (512MB RAM shared).

Xbox One (Durango)

Xbox One
Minecraft.Client/Durango/ — Codename: Durango

Key Features

  • Enhanced rendering capabilities over Xbox 360
  • Xbox Live integration (modern APIs)
  • Improved multiplayer and social features
  • Higher resolution assets (1080p)
  • Background download for DLC
  • Libraries: Durango/4JLibs/, Durango/Miles/, Durango/Iggy/, Durango/Sentient/
  • Media: DurangoMedia/ for platform-specific assets

PlayStation 3

PlayStation 3
Minecraft.Client/PS3/
  • Target: PlayStation 3 console

Key Features

  • Cell processor SPU task support (parallel processing on SPUs)
  • Edge libraries integration (Sony's performance optimization toolkit)
  • Boost 1.53 dependency (PS3/PS3Extras/boost_1_53_0/)
  • DirectX compatibility layer (PS3/PS3Extras/DirectX/)
  • HeapInspector for memory debugging (PS3/PS3Extras/HeapInspector/)
  • Trophy system integration
  • Sony Network Platform integration
  • Libraries: PS3/4JLibs/, PS3/Miles/, PS3/Iggy/, PS3/Sentient/
  • Media: PS3Media/, PS3_GAME/, TROPDIR/ (trophies)
Notes: Most technically challenging port due to Cell architecture. SPU tasks offload work from main PPU.

PlayStation 4 (Orbis)

PlayStation 4
Minecraft.Client/Orbis/ — Codename: Orbis

Key Features

  • Modern console hardware (x86-64 architecture)
  • Enhanced rendering and higher resolution support
  • PlayStation Network integration
  • Trophy system
  • Sentient middleware integration
  • Share button/streaming support
  • Libraries: Orbis/4JLibs/, Orbis/Miles/, Orbis/Iggy/, Orbis/Sentient/
  • Media: OrbisMedia/, PS4_GAME/, sce_sys/

PlayStation Vita

PlayStation Vita
Minecraft.Client/PSVita/
  • Target: PlayStation Vita handheld

Key Features

  • Touch input support (front and rear touchpads)
  • Custom UI for small 5" OLED screen (544p native)
  • Vita-specific SWF media at 480p resolution
  • Deferred chunk building with critical chunk marking (performance optimization)
  • Reduced render distance and world size
  • Memory-optimized chunk compression
  • Wi-Fi multiplayer support
  • Libraries: PSVita/4JLibs/, PSVita/Miles/, PSVita/Iggy/, PSVita/Sentient/
  • Media: PSVitaMedia/
Notes: Most memory-constrained platform (512MB total). Aggressive optimization required: deferred chunk building, critical chunk prioritization, reduced world height.

Shared Network Code

  • Common/Network/Sony/ — Shared Sony platform networking code (PS3, PS4, Vita)
  • Platform-specific network managers handle matchmaking, lobby management, and NAT traversal
  • LAN discovery used for LCEMP Windows multiplayer
