Audio System

Overview

Miles Sound System integration for audio playback. ConsoleSoundEngine provides the abstract interface, SoundEngine is the concrete implementation. Supports 3D positional audio with per-player listeners for split-screen. Streaming music system with state machine handles background tracks per dimension.

Stream State Machine (MUSIC_STREAMSTATE)

The music streaming system uses a state machine to manage track lifecycle. The states and their transitions are:

StateDescriptionTransitions To
IdleNo music is active. The system waits for a music delay timer to expire before selecting and starting a new track.Stop
StopA request to stop the current stream has been issued. Prepares the stream for shutdown.Stopping
StoppingThe stream is fading out or finalizing. Waits for the audio driver to confirm the stream has stopped.Opening, Idle
OpeningA new track is being opened on a background thread (m_openStreamThread). The stream file is loaded and decoded.Play, OpeningCancel
OpeningCancelA cancel was requested while a track was being opened. Waits for the background thread to finish before returning to idle.Idle
PlayThe stream has been opened successfully. Playback is initiated with the configured volume and position.Playing
PlayingThe track is actively playing. The system monitors for completion, volume changes, or stop requests.Completed, Stop
CompletedThe track has finished playing. The system cleans up the stream handle and returns to idle with a new music delay.Idle

Audio Data Structures

Core data structures used throughout the audio system for spatial positioning, listener configuration, and sound playback parameters.

struct AUDIO_VECTOR {
    float x;
    float y;
    float z;
};

struct AUDIO_LISTENER {
    bool        bValid;
    AUDIO_VECTOR vPosition;
    AUDIO_VECTOR vOrientFront;
};

struct AUDIO_INFO {
    float x;
    float y;
    float z;
    float volume;
    float pitch;
    int   iSound;
    bool  bIs3D;
    bool  bUseSoundsPitchVal;
};
StructPurpose
AUDIO_VECTORSimple 3D vector for audio positioning. Stores x, y, z float components used by the Miles Sound System for spatial calculations.
AUDIO_LISTENERRepresents a single audio listener in 3D space. Each split-screen player has one. Contains a validity flag, world position, and front-facing orientation vector for directional audio.
AUDIO_INFODescribes a sound to be played. Packs position, volume, pitch, sound ID, and flags indicating whether the sound is 3D positional and whether to use the sound's built-in pitch value.

Platform Audio

Each target platform (Windows64, Xbox, Durango, PS3, Orbis, PSVita) has its own audio initialization path and driver configuration. The Miles Sound System provides the cross-platform abstraction layer, exposing a unified API across all supported consoles and PC. Music files use the Binka audio format for compressed streaming playback. Sound bank management is handled per-platform, with each platform loading its own set of compiled sound banks at startup.
