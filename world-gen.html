World Generation

Layer-based biome generation system with noise-driven terrain. The chunk generation pipeline flows from BiomeSource through LevelSource to ChunkStorage. Three dimensions (Overworld, Nether, The End) each with unique generation. The features system handles structures and decorations.

Biome Infrastructure

Supporting classes that manage biome data generation, caching, decoration, and layer-based overrides.

BiomeSource World Gen

Generates biome data for chunk coordinates using the layer system. Acts as the entry point for all biome lookups during world generation — the LevelSource queries the BiomeSource to determine which biome occupies each column of blocks in a chunk.

BiomeDecorator World Gen

Places trees, flowers, ores, and structures within a biome. Called during the decoration pass after terrain shape is generated. Each biome can customize decoration counts and feature types through its decorator instance.

TheEndBiomeDecorator World Gen extends BiomeDecorator

Special decorator for The End dimension. Handles placement of obsidian pillars, end crystals, and the exit portal structure unique to the end island.

BiomeCache World Gen

Caches biome lookups for performance. Stores previously computed biome data so that repeated queries for the same chunk coordinates do not recompute the full layer pipeline. Cache entries are evicted when memory pressure requires it.

FixedBiomeSource World Gen extends BiomeSource

Returns a single fixed biome for all coordinate queries. Used for superflat worlds and test/debug scenarios where uniform biome data is desired.

BiomeInitLayer World Gen

Initial biome assignment layer in the layer pipeline. Produces the first rough biome map by assigning biome IDs based on noise values and climate zones. Subsequent layers refine this output.

BiomeOverrideLayer World Gen

Overrides biome assignments from previous layers. Used in tutorial worlds and other special world types to force specific biomes into designated regions regardless of normal generation rules.

Level Sources

Level source classes are responsible for generating the actual terrain data for each chunk. Each world type and dimension has a dedicated level source implementation.

RandomLevelSource Overworld

Standard overworld terrain generation. Produces natural terrain using multi-octave noise, then populates chunks with caves, ravines, ore veins, dungeons, villages, strongholds, mineshafts, and other structures. The primary level source for survival worlds.

FlatLevelSource Overworld

Flat/superflat world generation. Produces uniform flat terrain with configurable layer composition. Uses a FixedBiomeSource to assign a single biome across the entire world. No caves, ores, or natural structures are generated by default.

CustomLevelSource Overworld

Custom world type generation. Extends the standard terrain pipeline with user-configurable parameters for noise scale, ore distribution, structure frequency, sea level, and biome size. Allows fine-grained control over world characteristics.

HellRandomLevelSource Nether

Nether dimension generation. Produces the characteristic netherrack terrain with a bedrock ceiling at y=127 and floor at y=0, lava lakes at y=31, and large open caverns. Generates nether fortresses and glowstone clusters. Uses HellBiome exclusively.

HellFlatLevelSource Nether

Flat nether generation. Produces a flat netherrack world for the nether dimension, analogous to FlatLevelSource for the overworld. Used in superflat nether configurations.

TheEndLevelRandomLevelSource The End

End dimension generation. Produces the central end stone island and surrounding void. Uses simplex noise to shape the island terrain. The ender dragon boss arena and obsidian pillars are placed during the decoration pass via TheEndBiomeDecorator.

Noise Generators

Noise generator classes produce the pseudo-random coherent noise values that drive terrain shape, cave carving, ore placement, and biome boundaries.

PerlinNoise Noise

Classic Perlin noise implementation used for terrain height generation, cave carving, and general-purpose noise sampling. Supports multi-octave fractal noise by layering multiple noise samples at different frequencies and amplitudes. The primary noise source for overworld terrain shape.

SimplexNoise Noise

Simplex noise variant that provides faster computation for higher-dimensional sampling. Produces smoother gradients than classic Perlin noise with fewer directional artifacts. Used in situations where performance is critical or where the noise output benefits from the simplex lattice structure.

ImprovedNoise Noise

Improved Perlin noise with better gradient selection. Uses an improved permutation table and gradient computation to eliminate visual artifacts present in the original Perlin noise algorithm. Provides higher-quality noise output at a slight performance cost.

Layer System

The layer system is the core mechanism for biome generation. Layers are applied sequentially to transform raw noise data into a fully resolved biome map. Each layer takes input from the previous layer and applies a specific transformation.

Pipeline: The layer chain begins with BiomeInitLayer, which assigns initial biome IDs, and passes through a series of transformation layers before producing the final biome data consumed by the terrain generator.

The layer pipeline handles the following operations:

  • Initial biome assignment — Assigns base biome IDs from noise values and climate parameters.
  • Biome edges — Smooths transitions between incompatible biomes (e.g., desert next to ice).
  • River generation — Carves river channels through the biome map by replacing biome IDs along computed river paths.
  • Zoom / scale — Upscales the biome map resolution, adding detail at each zoom level through interpolation.
  • Island generation — Introduces ocean and island patterns at the continental scale before biome assignment.

Each layer is a lightweight class that implements a getArea() method, returning an array of biome IDs for a given rectangular region. Layers are composed into a chain at world creation time, and the final layer in the chain is queried by BiomeSource during chunk generation.

Features

The Feature base class represents structures and decorations that are placed into the world after terrain shape generation. Features range from individual trees and ore veins to large multi-chunk structures like villages and strongholds.

  • Trees — Various tree types placed per biome (oak, birch, spruce, jungle, acacia). Each biome's getTreeFeature() method selects the appropriate type.
  • Caves — Connected cave systems carved through terrain using worm-like noise tunnels. Cave generation runs across chunk boundaries to produce seamless networks.
  • Dungeons — Small rooms containing a mob spawner and loot chests. Placed at random positions underground where open space exists.
  • Villages — Multi-building settlements generated on relatively flat terrain in plains, desert, and savanna biomes. Composed of houses, farms, wells, churches, and paths.
  • Strongholds — End portal structures generated deep underground. A fixed number of strongholds are placed per world at calculated positions radiating from the origin.
  • Mineshafts — Underground mining tunnel networks with wooden supports, rails, cave spider spawners, and loot chests. Generated as branching corridors at mid-depth.
  • Ores — Ore vein placement for coal, iron, gold, diamond, redstone, lapis lazuli, and emerald. Each ore type has configurable vein size, count per chunk, and height range.

Dimensions

The engine supports three distinct dimensions, each with its own level source, biome configuration, lighting model, and gameplay rules.

Overworld ID 0

The standard survival world. Features the full variety of biomes, day/night cycle with sun and moon, weather (rain and snow), and the complete set of structures (villages, strongholds, mineshafts, dungeons). Terrain height ranges from y=0 (bedrock) to y=256. Sea level is at y=63. Uses RandomLevelSource for terrain generation and BiomeSource with the full layer pipeline for biome assignment.

The Nether ID -1

The hell dimension. Characterized by netherrack terrain, lava oceans at y=31, a bedrock ceiling at y=127, and no sky or natural light. The compressed vertical space (128 blocks vs. 256) creates a claustrophobic environment. Features include nether fortresses, glowstone clusters, nether wart, and blaze/ghast/zombie pigman spawning. Uses HellRandomLevelSource and the single HellBiome.

The End ID 1

The end dimension. Consists of a central end stone island floating in void, surrounded by darkness. Home to the ender dragon boss fight with obsidian pillars and end crystals. No weather, no day/night cycle, and a single TheEndBiome. Uses TheEndLevelRandomLevelSource for terrain and TheEndBiomeDecorator for structure placement. Endermen are the only naturally spawning mob.
