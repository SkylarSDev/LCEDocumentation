Tiles & Items

Overview

The tile system represents all blocks in the world with a global registry of 4096 slots. Items represent all holdable objects with a 32000-slot registry. TileEntities add persistent data to specific block types. TileItems bridge the gap between items and placeable blocks.

Tile Base Class

Tile
Minecraft.World/Tile.h

Base class for all block types in the world. Each tile occupies a slot in the global tiles[4096] registry and defines the block's physical properties, rendering behavior, and interaction logic. Tiles are stateless singletons — per-position data is stored in the level's block/data arrays or in associated TileEntity instances.

TileEntity Classes

TileEntity
Minecraft.World/TileEntity.h

Base class for blocks that store additional data beyond tile ID and metadata. TileEntities are serialized with the chunk and ticked independently of the block's random tick. Each TileEntity holds its world position and a back-reference to the level. Subclasses implement load() and save() for NBT serialization, and optionally override tick() for per-frame logic.

Item Base Class

Item extends enable_shared_from_this<Item>
Minecraft.World/Item.h

Base class for all holdable objects. Items occupy slots in the global items[] array (up to 32000 entries). Unlike tiles, items are not placed directly in the world — they exist in inventories and as dropped entities. TileItem subclasses bridge the gap between items and placeable blocks.
