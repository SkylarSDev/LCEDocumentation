Storage System

Region-based save format (McRegion). LevelStorage interface with multiple implementations. Compressed tile/data/light storage for memory efficiency on consoles. Console save file system with split file support. NBT serialization via CompoundTag.

LevelStorage Interface

Storage Implementations

DirectoryLevelStorage Storage

File system directory-based storage. Stores world data in a directory tree on disk, with separate files for level data, player data, and region files.

McRegionLevelStorage Storage

McRegion format storage. Uses the standard Minecraft region file format where chunks are grouped into 32x32 region files for efficient disk I/O and reduced file handle usage.

MemoryLevelStorage Storage

In-memory storage for testing and temporary worlds. All data is held in memory with no disk persistence, useful for unit tests and throwaway world instances.

MockedLevelStorage Storage

Mock implementation for testing. Provides controllable storage behavior for automated test scenarios without requiring real file system access.

Chunk Storage

ChunkStorage Interface

Base interface for chunk persistence. Defines the contract for loading and saving individual chunks within a dimension.

McRegionChunkStorage Storage

Region file-based chunk storage. Reads and writes chunk data within McRegion format files, handling compression, sector allocation, and chunk coordinate mapping within each region file.

MemoryChunkStorage Storage

In-memory chunk storage. Stores chunk data in memory maps for testing and temporary world scenarios without disk I/O.

OldChunkStorage Storage

Legacy chunk format support. Handles reading chunks stored in the older per-chunk file format used before the region system was introduced.

ZonedChunkStorage Storage

Zone-based chunk organization. Groups chunks into zones for optimized loading and saving patterns, particularly useful for console memory management.

ChunkStorageProfileDecorator Storage

Performance profiling wrapper. Decorates any ChunkStorage implementation with timing instrumentation for measuring load/save performance characteristics.

LevelChunk

LevelChunk Storage

Compressed Storage

CompressedTileStorage Storage

Compresses tile ID data for memory efficiency. Critical for console hardware where RAM is severely constrained. Uses run-length or palette-based encoding to reduce the memory footprint of block storage within chunks.

SparseDataStorage Storage

Sparse storage for tile metadata. Only stores non-zero values, significantly reducing memory usage for chunks where most blocks have default metadata (e.g., stone, dirt, air).

SparseLightStorage Storage

Sparse storage for sky and block light values. Only stores non-default light values, reducing memory overhead for chunks with large uniform light regions (e.g., fully lit sky columns or dark underground areas).

Level Data

LevelData Storage

World metadata: seed, spawn position, time, weather, game mode, game rules. Persisted alongside chunk data and loaded when a world is opened.

LevelSettings Storage

World creation settings: seed, game mode, difficulty, features. Used when creating a new world to configure the initial generation parameters.

LevelSummary Storage

Brief world info for the save selection screen. Contains the world name, last played timestamp, game mode, and file size without loading the full level data.

LevelType Storage

World type enumeration: default, flat, amplified. Determines which level source and generation pipeline is used for terrain creation.

DerivedLevelData Storage

Derived and computed level data. Provides a view over base level data with additional computed properties, used in contexts where the full mutable level data is not needed.

Console Save System

ConsoleSaveFile Console

Console-specific save file wrapper. Encapsulates the platform-specific save file format used on console hardware, handling container formats and metadata.

ConsoleSaveFileConverter Console

Converts between save formats. Handles migration of save data between different console save format versions and cross-platform save conversion.

ConsoleSaveFileIO Console

I/O operations for console saves. Manages the low-level read and write operations for console save files, including buffering and error handling.

ConsoleSaveFileInputStream Console

Read stream for save data. Provides a sequential input stream interface for reading data from a console save file.

ConsoleSaveFileOutputStream Console

Write stream for save data. Provides a sequential output stream interface for writing data to a console save file.

ConsoleSaveFileOriginal Console

Original unmodified save format. Represents the initial save format before any format migrations or conversions have been applied.

ConsoleSaveFileSplit Console

Split save file support for large worlds. When a world exceeds the single-file size limit on console platforms, the save is split across multiple files for storage.

ConsoleSavePath Console

Save file path management. Resolves and manages file paths for console save data, handling platform-specific path conventions and storage locations.

SavedData Storage

Generic saved data container. Base class for any auxiliary data that needs to be persisted alongside the world (e.g., map data, scoreboard data, village data).

SavedDataStorage Storage

Storage manager for SavedData instances. Tracks, loads, and saves all registered SavedData objects, keyed by their string identifiers.

Region System

Region Storage

Represents a region of chunks. A region covers a 32x32 area of chunks and serves as the logical grouping unit for the McRegion file format.

RegionFile Storage

Single region file containing chunk data. Manages the on-disk file that stores up to 1024 chunks (32x32) with sector-based allocation, offset tables, and per-chunk compression.

RegionFileCache Storage

Cache for open region files. Maintains a pool of recently accessed RegionFile instances to avoid repeated file open/close overhead when accessing nearby chunks.

Level Storage Sources

LevelStorageSource Interface

Factory interface for creating LevelStorage instances. Abstracts the creation of storage backends so that the engine can work with different storage formats without hard-coded dependencies.

DirectoryLevelStorageSource Storage

Creates directory-based storage. Factory that produces DirectoryLevelStorage instances pointing at a given save directory on the file system.

McRegionLevelStorageSource Storage

Creates McRegion-based storage. Factory that produces McRegionLevelStorage instances using the region file format for chunk persistence.

MemoryLevelStorageSource Storage

Creates memory-based storage. Factory that produces MemoryLevelStorage instances for testing and temporary world scenarios.

Map Data

MapItemSavedData Storage

Per-player map item data. Stores the explored areas, pixel colors, and markers for each in-game map item. Persisted as SavedData and keyed by the map's unique numeric ID. Tracks which portions of the world each player has revealed on their map copies.
